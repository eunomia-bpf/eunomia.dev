# 每日内容产出计划说明

> 状态：内部执行草案，不发布。创建日期：2026-07-19。

本目录按月份保存每日运营计划。每天不是一个“一键任务”，而是一个小型工作包：按 `.agents/skills/eunomia-content-patrol/SKILL.md` 执行真实的内容巡检、发布、转发或新闻整理，不为了交差产出无意义 artifact。

## 执行顺序

1. 打开当前月份文件，例如 `draft/plan/2026-07.zh.md`。
2. 找到当天日期下面的 list。
3. 确认昨天是否留下阻塞或待确认项。
4. 完成当天最有意义的任务：计划发布、平台刷帖与高价值转发、每日新闻报告、ledger 校对或平台状态推进。
5. 如果涉及平台真实操作，只用侧边栏/应用内浏览器，并按 `eunomia-content-patrol` 的 standing authorization 和停止条件执行。
6. 更新 `draft/content-daily-log-YYYY-MM.md` 或当月计划里的完成记录。
7. 如确认了平台状态，更新 `.github/publisher/media/platforms/*.json`、`published.md` 或 `not-published.md`。

## 计划文件

- `2026-07.zh.md`
- `2026-08.zh.md`

## 每日完成标准

- 至少完成一个有意义的动作：真实发布、1-2 个高价值 quote/repost、每日新闻/雷达报告、计划内平台草稿进入可发布状态、ledger 状态确认或明确跳过理由。
- 不把 daily log、空草稿、形式化截图当作当天核心产出。
- 当天计划里至少一项任务有明确进展；能推进两项时再推进两项。
- 明天的第一步动作已经写清楚。
