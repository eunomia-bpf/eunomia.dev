Introducing AgentNebula: See days or weeks of your AI Agent's work in 30 seconds.

In many of my software-development and auto-research experiments, I start with a simple goal or idea and let an Agent iterate inside a repo for two or three days.

When I return, I want to understand the run quickly: what it changed, how the work evolved, whether it tested before writing code, which modules became hotspots, and how its activity was divided among code, papers, tests, and documentation.

That is why I built AgentNebula in AgentSight.

AgentNebula replays the evolution of a codebase from the Agent's real file activity. Files are stars, directories form stable color clusters, and reads briefly light up and enlarge the files being accessed. Creates, renames, and deletes reshape the star field, while Git commits appear as flashes around the outer frame.

The result turns a long, difficult-to-read event history into a visual record of how the software grew: where the Agent's attention moved, where repeated work accumulated, and which areas were left behind and later revisited.

As vibe coding becomes more common, more code is being written while the human is away. I want that process to remain reviewable: what the Agent tried, why something failed, what it learned, and whether it avoided the same mistake later.

It also gives me a way to examine how I design skills and harnesses. Did the Agent create documents it never read again? Did it spend most of its time churning through low-value tests instead of improving the code?

This animation follows ACTplane across multiple Agent sessions.

Run it locally:

`agentsight vis`
