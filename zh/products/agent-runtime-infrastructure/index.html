旗舰 · AI Agent 可观测与执行控制

看清 AI agent 在机器上真正做了什么

AI agent 会运行命令、改写文件、启动进程并访问远程服务，但当一次运行失败、卡住或行为异常时，应用日志往往解释不了机器上真正发生了什么。AgentSight 是本地优先的 AI agent top / strace，把 prompt、模型调用和工具决策连接到进程、文件、网络与资源行为；ActPlane 在这些系统边界上执行运行时策略。

本地优先无需 SDK / proxy支持 closed-source CLI评估中 CPU 开销低于 3%
开源采用AgentSight
文档GitHub讨论 pilot

01

实时看见

查看活跃 session、进程、模型与工具调用，以及文件、网络和资源活动。

02

解释失败

把 prompt、skill 和工具决策关联到错误与系统效果，定位卡住、重试和异常行为。

03

定位成本

发现慢步骤、重复模型或工具调用、token-heavy session 和高资源进程。

04

审计影响

确认访问过哪些服务、修改过哪些文件，并保留可查询、可回放的执行记录。

应用 trace 之外发生了什么

应用层工具适合在你拥有应用代码时观测 prompt、token、eval 和 latency。AgentSight 补上它们经常缺失的系统边界：从现有二进制和 closed-source CLI 外部观测，并把 LLM 流量与进程、文件和网络事件关联起来。

应用层 / SDK / gateway

  • 每个应用需要接入 SDK、callback 或 gateway
  • closed-source CLI 只能依赖它主动暴露的日志
  • trace 通常停在 framework 或 process 边界
  • 容易漏掉 subprocess 和本地文件活动

AgentSight · 系统层

  • ✓从进程外部观测，无需修改 agent 代码
  • ✓适用于现有二进制和 closed-source CLI
  • ✓关联 LLM、process、file 和 network 事件
  • ✓本地记录，并可导出 OpenTelemetry GenAI spans

当应用日志讲不清一次 agent 运行

从真实触发事件出发，选择需要解释、接入或控制的系统行为。

运行失败或行为异常

调试一次 coding agent session

模型和工具日志显示调用完成，但你仍需要知道它启动了什么进程、改了什么文件、访问了什么服务。

Platform / SRE

接入现有 observability 平台

无需逐个修改 agent，把 closed-source CLI 和不同框架的系统行为导出到已有 tracing 或 profiling 管线。

Security / governance

审计并约束系统影响

先用 AgentSight 取得进程、文件和网络证据，再由 ActPlane 在 exec、file、network 和 syscall 边界执行策略。

组成

AgentSight 负责看见和解释真实系统行为；ActPlane 负责在同一组系统边界上执行控制。两者职责明确，也可以独立采用。

Observe & explain

AgentSight

本地优先的 AI agent profiling 与 monitoring，把 prompt、模型和工具决策关联到进程、文件、网络与资源行为，并支持 report、agentpprof 和 OpenTelemetry 导出。

文档GitHub研究文章

Harness

ActPlane

OS-level harness，在 syscall、exec、file 和 network 边界用系统级策略约束 agent 行为，并提供应用层日志之外的执行证据。
