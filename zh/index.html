Eunomia 提供什么

Eunomia 的工作集中在 runtime infrastructure、公开资源和 AI agent systems。

Runtime extension

让 eBPF 不只停留在内核工作流里

通过 userspace execution、GPU paths、JIT/AOT 支持和部署模型，把 eBPF 扩展到生产工具链可以直接使用的系统路径。

Resources

发布可复用的教程、文档、博客和论文

面向 eBPF 开发者维护可执行教程、文档入口、技术文章和研究产物。

AI Agents

在系统层观测并约束 AI agent

零插桩、框架无关地看清 agent 真正做了什么，并对它能做什么执行运行时策略。

Runtime

bpftime

面向 userspace 的高性能 eBPF runtime 和扩展框架，覆盖 uprobe、USDT、syscall、XDP、GPU 等工作流。

文档GitHubOSDI 2025

GPU

gpu_ext

通过动态 PTX 注入把 eBPF 的可编程性与可观测性扩展到 GPU，是 bpftime 的一部分。

GPU 文档LPC 2025

Runtime core

llvmbpf

支持 LLVM JIT/AOT 的 userspace eBPF VM，也是 bpftime 的编译和执行核心。

文档GitHub博客

工具链

eunomia-bpf

使用 JSON metadata 和 WebAssembly OCI workflow 构建、打包、分发并运行 eBPF 程序。

文档GitHubApsara 2022

WebAssembly

wasm-bpf

基于 WebAssembly 的 eBPF 工具链和运行时支持，适合轻量级沙箱和云原生部署场景。

文档GitHubKubeCon NA 2023

AI Agents

Agent observability

AgentSight

Agent policy

ActPlane

面向 AI agent 的 OS-level safety、security 和 governance 控制，用 policy-as-code 约束进程树、文件、网络影响、数据流标签和工作流门禁。

文档示例GitHub
