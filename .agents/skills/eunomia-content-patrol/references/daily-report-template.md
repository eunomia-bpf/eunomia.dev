# Daily Report Template

Use this template for a compact daily news/report brief when no planned publish
or high-value repost is better.

## File Location

Default path:

`draft/media/YYYY-MM-DD/daily-radar.md`

Use a source-specific `draft/media/YYYY-MM-DD/<slug>/` folder when the report is
part of a planned launch or follow-up.

## Format

```markdown
# Daily Radar - YYYY-MM-DD

## Published Or Reposted

- Platform:
- URL:
- Why it mattered:
- Ledger update:

## High-Signal Items

- [Title](URL)
  Why it matters:
  Possible Eunomia response:

- [Title](URL)
  Why it matters:
  Possible Eunomia response:

## Skip Decisions

- Item:
  Reason skipped:

## Next Action

- Concrete next step:
```

## Quality Bar

- Prefer 3-5 high-signal items over broad aggregation.
- Use primary sources when available: repo, paper, release note, official docs,
  public post, public talk page, or project page.
- Do not quote long passages. Summarize in original words and link sources.
- Do not include private strategy, customer claims, pricing, unreleased roadmap,
  or unverifiable numbers.
