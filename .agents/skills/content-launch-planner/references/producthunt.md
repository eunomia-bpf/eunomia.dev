# Product Hunt

Use this reference only when Product Hunt is being considered. Product Hunt is
not a default destination for every eunomia.dev post.

Official references to re-check before a real submission:

- https://www.producthunt.com/launch
- https://www.producthunt.com/launch/preparing-for-launch
- https://www.producthunt.com/launch/before-launch

## Fit Check

Consider Product Hunt when the artifact is a tryable product/tool, not just a
blog post or paper summary.

Good candidates:

- public GitHub project with clear install/use path
- hosted demo or landing page
- AI agent tool, developer tool, observability/security product, workflow tool
- meaningful new iteration of an existing product
- team bandwidth to answer comments during launch day

Weak candidates:

- pure research commentary
- a tutorial without a product artifact
- repo with unclear onboarding
- no screenshot/gallery/demo
- no person available to respond to questions

## Official Rules And Fields

Based on Product Hunt's launch guide, verify these current details before
submission:

- A maker can hunt their own product; a third-party hunter is not required.
- Company accounts are prohibited; use a real maker account.
- A rule of thumb is 12:01 AM Pacific Time for a full 24-hour homepage cycle,
  but Product Hunt also says the best launch time depends on readiness and goals.
- Do not directly ask people to upvote. Ask people to visit, comment, and share
  honest feedback.
- Product URL should be the primary product page; in some cases an external URL
  such as a GitHub repo is acceptable. Shortened links and tracking links are
  not accepted.
- Name should be the product name only, with no description or emoji unless the
  emoji is part of the legitimate name.
- Tagline is max 60 characters and should plainly explain the product.
- Description is max 500 characters and should state the value proposition and
  key features.
- Choose up to 3 launch tags that strongly match the product.
- Thumbnail should be square, recommended 240x240, under 3 MB.
- Gallery needs at least 2 images before the post page is viewable; recommended
  image size is 1270x760.
- Video is optional. Product Hunt supports YouTube links and the video must not
  be private.
- First maker comment is important and should start the discussion.
- A launch can be scheduled up to 1 month in advance.

## Product Hunt Plan Fields

When Product Hunt is included, produce:

- Fit verdict: include, maybe later, or skip.
- Product URL recommendation: landing page, GitHub repo, docs, or demo.
- Name and 60-character tagline options.
- Description draft under 500 characters.
- Up to 3 tag candidates.
- Thumbnail/gallery/demo asset needs.
- Maker comment outline: identity, why built, problem, solution, differentiator,
  feedback ask.
- Safe outreach wording: "visit", "comment", "feedback", never "upvote".
- Launch-day support plan: who answers comments, when to check, what issues to
  escalate.
- Post-launch follow-up: thank commenters, collect feedback themes, update repo
  docs, publish a retrospective only if there is something useful to share.

## Safe Messaging

Allowed:

- "Take a look and tell us what you think."
- "We would value technical feedback."
- "If this is useful, share it with someone building similar systems."
- "What would make this more useful for your workflow?"

Avoid:

- "Please upvote."
- "Help us reach #1."
- "Vote for us."
- "Upvote to get a reward."
- Copy-pasted comments or mass outreach to strangers.

## Browser Boundary

Use the Product Hunt website in a normal browser session for real preparation,
preview, submission, screenshots, and comment follow-up. Do not use hidden APIs
or automation to submit, vote, comment, message, or scrape private state.
